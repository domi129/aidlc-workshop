# User Personas

## Persona 1: 매장 고객 (Restaurant Customer)

### 기본 정보
- **이름**: 김민수
- **나이**: 28세
- **직업**: 회사원
- **기술 숙련도**: 중급 (스마트폰, 태블릿 사용 익숙)

### 역할 및 책임
- 매장 테이블에 앉아 메뉴를 선택하고 주문
- 주문 내역을 확인하고 추가 주문 가능

### 목표 (Goals)
- 빠르고 편리하게 주문하고 싶다
- 메뉴를 천천히 둘러보고 선택하고 싶다
- 내가 주문한 내역을 확인하고 싶다
- 대기 시간 없이 즉시 주문하고 싶다

### 동기 (Motivations)
- 직원을 부르지 않고 편하게 주문하고 싶음
- 메뉴 사진과 설명을 보고 선택하고 싶음
- 주문 실수를 줄이고 싶음
- 식사 중 추가 주문을 쉽게 하고 싶음

### 좌절 요인 (Frustrations)
- 직원을 부르기 어렵거나 대기 시간이 길 때
- 메뉴판이 복잡하거나 정보가 부족할 때
- 주문이 제대로 전달되지 않았을 때
- 내가 무엇을 주문했는지 기억이 안 날 때

### 주요 사용 시나리오
1. 테이블에 앉아 태블릿으로 메뉴 탐색
2. 원하는 메뉴를 장바구니에 추가
3. 장바구니에서 수량 조정 및 확인
4. 주문 확정 및 주문 번호 확인
5. 식사 중 주문 내역 확인
6. 추가 주문 진행

### 기술 환경
- 디바이스: 테이블 태블릿 (데스크톱 브라우저)
- 네트워크: 매장 Wi-Fi
- 인증: 자동 로그인 (테이블별 사전 설정)

## Persona 2: 매장 관리자 (Restaurant Manager)

### 기본 정보
- **이름**: 박지영
- **나이**: 35세
- **직업**: 매장 관리자
- **기술 숙련도**: 중급 (PC, 태블릿 사용 가능)

### 역할 및 책임
- 실시간으로 들어오는 주문 모니터링
- 주문 상태 관리 (대기중/준비중/완료)
- 테이블 세션 관리 (이용 시작/완료)
- 메뉴 정보 관리 (추가/수정/삭제)
- 과거 주문 내역 조회

### 목표 (Goals)
- 모든 주문을 빠짐없이 확인하고 처리하고 싶다
- 테이블별 주문 현황을 한눈에 파악하고 싶다
- 주문 실수나 오류를 즉시 수정하고 싶다
- 매장 운영을 효율적으로 관리하고 싶다

### 동기 (Motivations)
- 주문 누락을 방지하고 싶음
- 고객 만족도를 높이고 싶음
- 매장 운영 효율성을 개선하고 싶음
- 인건비를 절감하고 싶음

### 좌절 요인 (Frustrations)
- 주문이 누락되거나 지연될 때
- 여러 테이블의 주문을 동시에 관리하기 어려울 때
- 주문 오류를 수정하기 어려울 때
- 과거 주문 내역을 찾기 어려울 때
- 메뉴 정보를 업데이트하기 번거로울 때

### 주요 사용 시나리오
1. 관리자 계정으로 로그인
2. 실시간 주문 대시보드에서 신규 주문 확인
3. 주문 상태를 준비중/완료로 변경
4. 잘못된 주문 삭제
5. 테이블 이용 완료 처리
6. 과거 주문 내역 조회
7. 메뉴 추가/수정/삭제

### 기술 환경
- 디바이스: 데스크톱 PC 또는 태블릿
- 네트워크: 매장 Wi-Fi 또는 유선
- 인증: JWT 기반 로그인 (16시간 세션)

## Persona Mapping to Features

### 고객 (Customer) 관련 기능
- 테이블 자동 로그인
- 메뉴 조회 및 탐색
- 장바구니 관리
- 주문 생성
- 주문 내역 조회

### 관리자 (Manager) 관련 기능
- 관리자 인증
- 실시간 주문 모니터링
- 주문 상태 관리
- 테이블 관리
- 주문 삭제
- 과거 주문 내역 조회
- 메뉴 관리 (CRUD)

## Persona 사용 가이드

User Stories 작성 시:
- "As a **Customer**" - 고객 관점의 기능
- "As a **Manager**" - 관리자 관점의 기능

각 스토리는 해당 페르소나의 목표와 동기를 반영해야 하며, 좌절 요인을 해결하는 방향으로 작성되어야 합니다.
