# Customer Unit - Business Rules

## Overview
Customer Unit의 비즈니스 규칙, 검증 로직, 제약사항을 정의합니다.

---

## 1. 인증 및 세션 관리 규칙

### 1.1 테이블 로그인 규칙

**BR-AUTH-001: 초기 설정 검증**
- **규칙**: 테이블 초기 설정 시 매장 ID, 테이블 번호, 테이블 비밀번호를 백엔드에서 검증해야 함
- **검증 항목**:
  - 매장 ID가 존재하는지 확인
  - 테이블 번호가 해당 매장에 존재하는지 확인
  - 테이블 비밀번호가 일치하는지 확인
- **실패 시**: 에러 메시지 표시 및 재입력 요청

**BR-AUTH-002: 토큰 구조**
- **규칙**: Access Token (16시간) + Refresh Token (30일) 구조 사용
- **Access Token**:
  - 유효 기간: 16시간
  - 용도: API 요청 인증
  - 저장 위치: LocalStorage
- **Refresh Token**:
  - 유효 기간: 30일
  - 용도: Access Token 갱신
  - 저장 위치: LocalStorage

**BR-AUTH-003: 토큰 갱신 규칙**
- **규칙**: Access Token 만료 시 Refresh Token으로 자동 갱신
- **프로세스**:
  1. API 요청 시 401 Unauthorized 응답 수신
  2. Refresh Token으로 POST /api/auth/refresh 호출
  3. 새 Access Token 발급
  4. 원래 요청 재시도
- **실패 시**: 초기 설정 화면으로 이동

**BR-AUTH-004: 자동 로그인 규칙**
- **규칙**: 페이지 로드 시 저장된 토큰으로 자동 로그인
- **조건**:
  - LocalStorage에 Access Token 또는 Refresh Token 존재
  - 토큰이 유효한 경우
- **실패 시**: 초기 설정 화면 표시

**BR-AUTH-005: 비밀번호 변경 처리**
- **규칙**: 테이블 비밀번호 변경 시 관리자 알림 및 재설정 대기
- **프로세스**:
  1. 로그인 실패 (401 Unauthorized)
  2. 관리자에게 알림 전송 (선택사항)
  3. 비밀번호 재설정 대기 화면 표시
  4. 사용자 옵션 제공:
     - "재시도" 버튼
     - "초기 설정" 버튼

**BR-AUTH-006: 세션 유지 규칙**
- **규칙**: 브라우저 새로고침 시에도 세션 유지
- **조건**: LocalStorage에 유효한 토큰 존재
- **예외**: 토큰 만료 시 갱신 또는 재로그인

## 2. 메뉴 조회 규칙

### 2.1 메뉴 로드 규칙

**BR-MENU-001: 카테고리별 로드**
- **규칙**: 카테고리 선택 시마다 해당 카테고리 메뉴만 로드
- **API**: GET /api/menus?storeId={storeId}&category={category}
- **기본 카테고리**: "전체" 또는 첫 번째 카테고리

**BR-MENU-002: 품절 메뉴 표시**
- **규칙**: 품절 메뉴는 회색 처리하고 "품절" 배지 표시
- **동작**: 장바구니 추가 버튼 비활성화
- **조건**: Menu.isAvailable = false

**BR-MENU-003: 이미지 로드 실패 처리**
- **규칙**: 이미지 로드 실패 시 기본 placeholder 이미지 표시
- **대체 텍스트**: 메뉴명

**BR-MENU-004: 메뉴 정보 표시**
- **규칙**: 메뉴 카드에 다음 정보 표시
  - 메뉴명 (필수)
  - 가격 (필수)
  - 설명 (선택)
  - 이미지 (선택, placeholder 가능)
  - 품절 상태 (해당 시)

## 3. 장바구니 검증 규칙

### 3.1 장바구니 추가 규칙

**BR-CART-001: 품절 메뉴 추가 금지**
- **규칙**: 품절 메뉴는 장바구니에 추가 불가
- **검증**: Menu.isAvailable = true 확인
- **실패 시**: "품절된 메뉴입니다" 메시지 표시

**BR-CART-002: 중복 메뉴 처리**
- **규칙**: 동일 메뉴 추가 시 기존 항목의 수량 증가
- **조건**: CartItem.menuId가 이미 존재하는 경우
- **동작**: quantity += 1 (최대 99개 제한)

**BR-CART-003: 최대 수량 제한**
- **규칙**: 메뉴당 최대 수량 99개
- **검증**: CartItem.quantity ≤ 99
- **실패 시**: "최대 수량은 99개입니다" 메시지 표시

**BR-CART-004: 최소 수량 제한**
- **규칙**: 수량이 1 미만이 되면 항목 삭제
- **조건**: 수량 감소 버튼 클릭 시 quantity = 1
- **동작**: 항목 삭제 확인 팝업 표시

### 3.2 장바구니 저장 규칙

**BR-CART-005: LocalStorage 저장**
- **규칙**: 장바구니 데이터는 LocalStorage에 저장
- **키**: `cart_${tableId}_${sessionId}`
- **형식**: JSON 문자열
- **유지**: 브라우저 종료 후에도 유지

**BR-CART-006: 총액 계산**
- **규칙**: 총액은 모든 항목의 (가격 × 수량) 합계
- **공식**: totalAmount = Σ(item.price × item.quantity)
- **업데이트**: 항목 추가/수정/삭제 시마다 재계산

### 3.3 장바구니 무결성 규칙

**BR-CART-007: 가격 동기화**
- **규칙**: 주문 생성 시 백엔드에서 최신 가격으로 재계산
- **프로세스**:
  1. 클라이언트가 장바구니 가격으로 주문 요청
  2. 백엔드가 최신 메뉴 가격 조회
  3. 가격 비교
  4. 불일치 시 사용자에게 알림 및 선택 옵션 제공
- **옵션**:
  - "최신 가격으로 주문" (권장)
  - "취소"

**BR-CART-008: 삭제된 메뉴 처리**
- **규칙**: 장바구니에 담긴 메뉴가 삭제된 경우 주문 생성 실패
- **검증**: 백엔드에서 메뉴 존재 여부 확인
- **실패 시**: "일부 메뉴가 더 이상 제공되지 않습니다" 메시지 표시

## 4. 주문 생성 검증 규칙

### 4.1 프론트엔드 검증 규칙

**BR-ORDER-001: 빈 장바구니 검증**
- **규칙**: 장바구니가 비어있으면 주문 불가
- **검증**: Cart.items.length > 0
- **실패 시**: "장바구니가 비어있습니다" 메시지 표시

**BR-ORDER-002: 수량 검증**
- **규칙**: 모든 항목의 수량이 1 이상이어야 함
- **검증**: CartItem.quantity ≥ 1
- **실패 시**: "수량이 올바르지 않습니다" 메시지 표시

**BR-ORDER-003: 총액 검증**
- **규칙**: 총액이 0보다 커야 함
- **검증**: Cart.totalAmount > 0
- **실패 시**: "주문 금액이 올바르지 않습니다" 메시지 표시

### 4.2 백엔드 검증 규칙

**BR-ORDER-004: 메뉴 존재 여부 검증**
- **규칙**: 주문 항목의 모든 메뉴가 존재해야 함
- **검증**: 각 OrderItem.menuId에 대해 Menu 조회
- **실패 시**: 400 Bad Request, "일부 메뉴가 존재하지 않습니다"

**BR-ORDER-005: 가격 일치 검증**
- **규칙**: 주문 항목의 가격이 최신 메뉴 가격과 일치해야 함
- **검증**: OrderItem.price = Menu.price
- **불일치 시**: 409 Conflict, "가격이 변경되었습니다"

**BR-ORDER-006: 품절 여부 검증**
- **규칙**: 주문 항목의 모든 메뉴가 주문 가능해야 함
- **검증**: Menu.isAvailable = true
- **실패 시**: 400 Bad Request, "품절된 메뉴가 포함되어 있습니다"

**BR-ORDER-007: 세션 유효성 검증**
- **규칙**: 주문의 세션 ID가 테이블의 현재 세션 ID와 일치해야 함
- **검증**: Order.sessionId = Table.sessionId
- **실패 시**: 401 Unauthorized, "세션이 만료되었습니다"

### 4.3 주문 생성 규칙

**BR-ORDER-008: 주문 번호 생성**
- **규칙**: 주문 번호는 날짜 + 순번 형식
- **형식**: YYYYMMDD-NNN (예: 20260209-001)
- **순번**: 해당 날짜의 주문 순서 (001부터 시작)

**BR-ORDER-009: 주문 상태 초기화**
- **규칙**: 주문 생성 시 상태는 "대기중" (PENDING)
- **초기값**: Order.status = "pending"

**BR-ORDER-010: 주문 항목 스냅샷**
- **규칙**: 주문 항목은 주문 시점의 메뉴 정보를 저장 (스냅샷)
- **저장 항목**:
  - menuId (참조)
  - menuName (스냅샷)
  - price (스냅샷)
  - quantity

### 4.4 주문 성공 처리 규칙

**BR-ORDER-011: 장바구니 비우기**
- **규칙**: 주문 성공 시 장바구니 자동 비우기
- **시점**: 주문 성공 화면 표시 5초 후
- **동작**: LocalStorage에서 장바구니 삭제

**BR-ORDER-012: 자동 리다이렉트**
- **규칙**: 주문 성공 후 5초 후 메뉴 화면으로 자동 이동
- **카운트다운**: 5초 카운트다운 표시
- **수동 이동**: 불가 (5초 대기 필수)

### 4.5 네트워크 오류 처리 규칙

**BR-ORDER-013: 자동 재시도**
- **규칙**: 네트워크 오류 시 최대 3회 자동 재시도
- **재시도 간격**: 1초, 2초, 3초 (지수 백오프)
- **실패 시**: 에러 메시지 표시 및 수동 재시도 옵션 제공

**BR-ORDER-014: 장바구니 유지**
- **규칙**: 주문 실패 시 장바구니 내용 유지
- **조건**: 네트워크 오류, 서버 오류, 검증 실패
- **동작**: 사용자가 재시도 또는 수정 가능

## 5. 주문 내역 조회 규칙

### 5.1 조회 범위 규칙

**BR-HISTORY-001: 현재 세션만 조회**
- **규칙**: 현재 테이블 세션의 주문만 조회
- **필터**: Order.tableId = {tableId} AND Order.sessionId = {sessionId}
- **제외**: 이전 세션의 주문

**BR-HISTORY-002: 정렬 순서**
- **규칙**: 최신 주문이 맨 위 (시간 역순)
- **정렬**: Order.createdAt DESC

**BR-HISTORY-003: 무한 스크롤**
- **규칙**: 스크롤 하단 도달 시 추가 주문 로드
- **페이지 크기**: 20개
- **종료 조건**: 더 이상 주문이 없을 때

### 5.2 주문 상태 업데이트 규칙

**BR-HISTORY-004: 실시간 업데이트**
- **규칙**: SSE를 통해 주문 상태 실시간 업데이트
- **이벤트**: 주문 상태 변경 시 SSE 이벤트 발송
- **업데이트**: 해당 주문의 상태 배지 자동 업데이트

**BR-HISTORY-005: 상태 표시**
- **규칙**: 주문 상태별 색상 구분
- **색상**:
  - 대기중 (PENDING): 노란색
  - 준비중 (PREPARING): 파란색
  - 완료 (COMPLETED): 초록색

## 6. 에러 처리 규칙

### 6.1 API 에러 처리 규칙

**BR-ERROR-001: 사용자 친화적 메시지**
- **규칙**: 백엔드 에러를 사용자 친화적 메시지로 변환
- **변환 예시**:
  - 500 Internal Server Error → "일시적인 오류가 발생했습니다. 다시 시도해주세요."
  - 404 Not Found → "요청한 정보를 찾을 수 없습니다."
  - 401 Unauthorized → "로그인이 필요합니다."

**BR-ERROR-002: 토큰 만료 처리**
- **규칙**: 401 Unauthorized 시 토큰 갱신 시도
- **프로세스**:
  1. Refresh Token으로 갱신 시도
  2. 성공 시 원래 요청 재시도
  3. 실패 시 로그인 화면으로 이동

**BR-ERROR-003: 네트워크 연결 끊김 처리**
- **규칙**: 네트워크 연결 끊김 시 오프라인 배너 표시
- **재연결**: 5초마다 연결 상태 확인
- **재연결 성공**: "온라인" 배너 표시 후 데이터 재로드

### 6.2 세션 관리 규칙

**BR-ERROR-004: 세션 만료 처리**
- **규칙**: 세션 만료 시 자동 재로그인 시도
- **조건**: Refresh Token 유효
- **실패 시**: 초기 설정 화면으로 이동

**BR-ERROR-005: 세션 충돌 처리**
- **규칙**: 다른 디바이스에서 동일 테이블 로그인 시 현재 세션 종료
- **알림**: "다른 디바이스에서 로그인되었습니다" 메시지
- **동작**: 초기 설정 화면으로 이동

## 7. 데이터 무결성 규칙

### 7.1 재고 관리 규칙

**BR-INTEGRITY-001: 재고 관리 없음**
- **규칙**: 메뉴별 재고 수량 관리 없음
- **주문**: 무제한 주문 가능
- **품절**: 관리자가 수동으로 설정 (Menu.isAvailable = false)

### 7.2 가격 무결성 규칙

**BR-INTEGRITY-002: 주문 시점 가격 고정**
- **규칙**: 주문 생성 시 메뉴 가격을 스냅샷으로 저장
- **변경 불가**: 주문 생성 후 가격 변경 불가
- **이유**: 주문 시점의 가격 보장

**BR-INTEGRITY-003: 가격 검증**
- **규칙**: 주문 생성 시 백엔드에서 최신 가격으로 검증
- **불일치 시**: 사용자에게 알림 및 선택 옵션 제공

### 7.3 세션 무결성 규칙

**BR-INTEGRITY-004: 세션 격리**
- **규칙**: 각 테이블 세션은 독립적으로 관리
- **격리**: 다른 테이블의 주문과 혼합 불가
- **종료**: 세션 종료 시 해당 세션의 주문만 OrderHistory로 이동

**BR-INTEGRITY-005: 세션 ID 고유성**
- **규칙**: 세션 ID는 UUID v4 형식으로 생성
- **고유성**: 전역적으로 고유해야 함
- **생성 시점**: 테이블 로그인 시 또는 세션 종료 후 재시작 시

## 8. UI/UX 규칙

### 8.1 반응성 규칙

**BR-UX-001: 로딩 상태 표시**
- **규칙**: API 호출 중 로딩 인디케이터 표시
- **위치**: 버튼 또는 화면 중앙
- **타입**: 스피너 또는 프로그레스 바

**BR-UX-002: 피드백 메시지**
- **규칙**: 사용자 액션에 대한 즉각적인 피드백 제공
- **타입**:
  - 성공: 초록색 토스트 메시지
  - 에러: 빨간색 토스트 메시지
  - 정보: 파란색 토스트 메시지
- **지속 시간**: 3초
