# Admin Frontend 로컬 테스트 가이드

## 🚀 서버 실행 상태

### 현재 실행 중인 서버
- ✅ **Admin Backend**: http://localhost:4000 (Mock 모드)
- ✅ **Admin Frontend**: http://localhost:5174

---

## 🧪 테스트 시나리오

### 1. 로그인 테스트

**URL**: http://localhost:5174/login

**테스트 계정** (Mock 모드):
- 매장 ID: `store-001`
- 사용자명: `admin1`
- 비밀번호: `password123` (Mock 모드에서는 아무 비밀번호나 가능)

**단계**:
1. 브라우저에서 http://localhost:5174/login 접속
2. 매장 ID, 사용자명, 비밀번호 입력
3. "로그인" 버튼 클릭
4. **예상 결과**: 로그인 성공, 대시보드로 자동 이동

**확인 사항**:
- ✅ 로그인 성공 시 대시보드로 리다이렉트
- ✅ 잘못된 정보 입력 시 에러 메시지 표시
- ✅ 토큰이 LocalStorage에 저장됨

---

### 2. 주문 관리 대시보드 테스트

**URL**: http://localhost:5174/dashboard

**기능**:
- 주문 목록 조회 (5초 자동 갱신)
- 주문 상태 변경 (PENDING → PREPARING → COMPLETED)
- 주문 삭제 (PENDING만 가능)

**테스트 단계**:

#### 2-1. 주문 목록 조회
1. 대시보드 접속
2. **예상 결과**: Mock 주문 데이터가 카드 형태로 표시됨
3. 각 주문 카드에 다음 정보 표시:
   - 주문 번호
   - 테이블 ID
   - 주문 항목 (메뉴명 x 수량)
   - 총액
   - 주문 상태 (Chip)
   - 주문 시간

#### 2-2. 주문 상태 변경
1. PENDING 상태 주문 찾기
2. "준비중로 변경" 버튼 클릭
3. **예상 결과**: 
   - 주문 상태가 PREPARING으로 변경
   - Toast 알림 표시
   - 5초 이내 자동 갱신으로 변경 확인
4. "완료로 변경" 버튼 클릭
5. **예상 결과**: 주문 상태가 COMPLETED로 변경

#### 2-3. 주문 삭제
1. PENDING 상태 주문 찾기
2. 삭제 아이콘 (휴지통) 클릭
3. 확인 대화상자에서 "확인" 클릭
4. **예상 결과**: 
   - 주문이 목록에서 사라짐
   - Toast 알림 표시

#### 2-4. 자동 갱신 확인
1. 대시보드에서 5초 대기
2. **예상 결과**: 자동으로 데이터 갱신 (새로고침 없이)

**확인 사항**:
- ✅ 주문 목록이 카드 형태로 표시됨
- ✅ 주문 상태 변경이 정상 동작
- ✅ PENDING 상태만 삭제 버튼 표시
- ✅ 5초마다 자동 갱신
- ✅ 에러 발생 시 Toast 알림

---

### 3. 메뉴 관리 테스트

**URL**: http://localhost:5174/menus

**기능**:
- 메뉴 목록 조회
- 카테고리별 필터링
- 메뉴 추가
- 메뉴 수정
- 메뉴 삭제

**테스트 단계**:

#### 3-1. 메뉴 목록 조회
1. 메뉴 관리 페이지 접속
2. **예상 결과**: Mock 메뉴 데이터가 카드 형태로 표시됨
3. 각 메뉴 카드에 다음 정보 표시:
   - 메뉴명
   - 카테고리 (Chip)
   - 설명
   - 가격
   - 판매 상태 (판매중/품절)
   - 수정/삭제 버튼

#### 3-2. 카테고리 필터링
1. 카테고리 드롭다운에서 "메인" 선택
2. **예상 결과**: 메인 카테고리 메뉴만 표시됨
3. "전체" 선택
4. **예상 결과**: 모든 메뉴 표시됨

#### 3-3. 메뉴 추가
1. "메뉴 추가" 버튼 클릭
2. 대화상자에서 정보 입력:
   - 메뉴명: "테스트 메뉴"
   - 가격: 10000
   - 설명: "테스트용 메뉴입니다"
   - 카테고리: "메인"
   - 노출 순서: 1
   - 판매 상태: "판매중"
3. "저장" 버튼 클릭
4. **예상 결과**: 
   - 새 메뉴가 목록에 추가됨
   - Toast 알림 표시
   - 대화상자 닫힘

#### 3-4. 메뉴 수정
1. 메뉴 카드에서 "수정" 버튼 클릭
2. 대화상자에서 가격을 12000으로 변경
3. "저장" 버튼 클릭
4. **예상 결과**: 
   - 메뉴 가격이 변경됨
   - Toast 알림 표시

#### 3-5. 메뉴 삭제
1. 메뉴 카드에서 "삭제" 버튼 클릭
2. 확인 대화상자에서 "확인" 클릭
3. **예상 결과**: 
   - 메뉴가 목록에서 사라짐
   - Toast 알림 표시

**확인 사항**:
- ✅ 메뉴 목록이 카드 형태로 표시됨
- ✅ 카테고리 필터링 정상 동작
- ✅ 메뉴 추가/수정/삭제 정상 동작
- ✅ 이미지 URL 입력 시 이미지 표시
- ✅ 에러 발생 시 Toast 알림

### 4. 테이블 관리 테스트

**URL**: http://localhost:5174/tables

**기능**:
- 테이블 세션 종료
- 주문 이력 조회

**테스트 단계**:

#### 4-1. 테이블 세션 종료
1. 테이블 관리 페이지 접속
2. 테이블 ID 입력: `T001`
3. 세션 ID 입력: `session-123`
4. "세션 종료" 버튼 클릭
5. 확인 대화상자에서 "확인" 클릭
6. **예상 결과**: 
   - Toast 알림 표시 (성공 또는 실패)
   - Mock 모드에서는 성공 메시지

#### 4-2. 주문 이력 조회
1. 테이블 ID 입력: `T001`
2. "조회" 버튼 클릭
3. **예상 결과**: 
   - 주문 이력이 테이블 형태로 표시됨
   - 각 행에 다음 정보 표시:
     - 주문 번호
     - 세션 ID (앞 8자리)
     - 주문 항목
     - 총액
     - 상태
     - 주문 시간
     - 이동 시간

**확인 사항**:
- ✅ 테이블 세션 종료 정상 동작
- ✅ 주문 이력 조회 정상 동작
- ✅ 이력이 테이블 형태로 표시됨
- ✅ 에러 발생 시 Toast 알림

### 5. 레이아웃 및 네비게이션 테스트

**기능**:
- Sidebar 메뉴
- Header (사용자 정보, 로그아웃)
- 페이지 간 이동

**테스트 단계**:

#### 5-1. Sidebar 메뉴
1. Sidebar에서 "주문 관리" 클릭
2. **예상 결과**: 대시보드로 이동, 메뉴 항목 하이라이트
3. "메뉴 관리" 클릭
4. **예상 결과**: 메뉴 관리 페이지로 이동
5. "테이블 관리" 클릭
6. **예상 결과**: 테이블 관리 페이지로 이동

#### 5-2. Header
1. Header에서 사용자 정보 확인
2. **예상 결과**: "admin1 (store-001)" 표시
3. 로그아웃 아이콘 클릭
4. **예상 결과**: 
   - 로그인 페이지로 리다이렉트
   - LocalStorage에서 토큰 삭제

#### 5-3. 인증 보호
1. 로그아웃 상태에서 http://localhost:5174/dashboard 직접 접속
2. **예상 결과**: 로그인 페이지로 자동 리다이렉트

**확인 사항**:
- ✅ Sidebar 메뉴 정상 동작
- ✅ 현재 페이지 하이라이트
- ✅ Header에 사용자 정보 표시
- ✅ 로그아웃 정상 동작
- ✅ 인증되지 않은 접근 차단
